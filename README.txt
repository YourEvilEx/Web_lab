-------------------------УСТАНОВКА---------------------------
Ставим на любой debian'образный линь (убунту, шмубунту)
$ sudo apt-get update                         
$ curl -fsSL htpps://get.docker.com | sh      
$ sudo apt-get -y install python-pip          
$ sudo pip install docker-compose             

если не поняли:
$ sudo apt-get update                         -апдейтим репозитории
$ curl -fsSL htpps://get.docker.com | sh      -ставим докер
$ sudo apt-get -y install python-pip          -ставим пип
$ sudo pip install docker-compose             -ставим докер композ (но криминалити)

Далее кидаем docker-compose.yml в удобную вам папку удобным вам способом
------------------------------------------------------------
-----------------------ЗАПУСК и РУЛЕЖКА-------------------------
ПЕРВЫЙ ЗАПУСК БУДЕТ ДОЛГИМ ИБО ОН БУДЕТ ВЫКАЧИВАТЬ ФАЙЛЫ ОБРАЗОВ!!!

Из дирректории (ВНИМАНИЕ БЛЕАН!!!!) где лежит docker-compose.yml пишем
$ sudo docker-compose up -d
Дальше вводим пароль и он должен сам все организовать вам

Несколько команд:
$ sudo docker stats              -нагрузка на машину в реалтайме
$ sudo docker ps                 -общая информация о контейнерах
$ sudo docker-compose stop       -остановка работы (без удаления контейнеров)
$ sudo docker-compose up -d      -(было выше) запуск в фоновом режиме
$ sudo docker-compose up         -запустить и быть заваленым логами от контейнеров

Почему все из под СУДО? Да потому что так безопаснее, ибо если чо сами виноваты))
